Principe:

  • Un esigate patché
  • Une extension esigate
  • Un esigate appli web J2EE
  • Un server portail
  • Un server webapp1
  • Un server webapp2
Chaque "application" (portail,webapp1,webapp2) a 1 page index qui contient un menu, et un fragment vignette. La page menu est sur le portail est agglomere les vignettes des 3 applications (avec try/except) + une quatrième qui correspond à un provider qui n'existe pas

Deployment : 4 tomcats

  1. Deployment [server] sur tomcat:8080 (server.xml modified : <Context .. path="/"/>)
  2. Deployment [portail] [webapp1] [webapp2] sur tomcat:9080
  3. Deployment [webapp1] sur tomcat:10080
  4. Deployment [webapp2] sur tomcat:11080

Details par module :

[esigate-core-patch)] :
jar contenant l'esigate-core patché
[extension] :
jar contenant l'extension esigate
[http] :
jar contenant le filtre de temporisation et le filtre de trace
[server] :
war esigate avec extension : paramétrage d'un timeout global 20s paramétrage d'un timeout sur les vignette 5s
[server] :
war esigate avec extension : paramétrage d'un timeout global 20s paramétrage d'un timeout sur les vignette 5s
[portail] :
war application web portail : page index + fragment vignette + fragment menu : filtre de trace
[webapp1] :
war application web 1 : page index + fragment vignette : filtre trace
[webapp2] :
war application web 2 : page index + fragment vignette : filtre de temporisation 10s sur chaque requete (index,vignette)+ filtre trace

Comportement attendu

=> Dans tous les pages index de portail et webapp2 on navigue sans voir le lien vers l'appli 1. Le temps de chargement des pages doit être de 5s (le temps du timeout sur la vignette de la webapp1).
=> Dans webapp1 : on doit pouvoir afficher la page en 10s mais sans la vignette
